# 玩家个性化档案管理 (Player Identity & Personalization)

这个技能允许你通过调用工具来“记住”并“调取”特定玩家的消息。这对于建立长期的、有深度的互动至关重要。

## 1. 核心操作流程 (Workflow)

1.  **先调取，再决策**：
    *   在对被提问的玩家做出决策（如考虑是否升级）之前，必须调用 `recall_player_memory`。
2.  **动态档案更新**：
    *   在产生实质性交互（如喜好宣告、行为贡献、状态变更）后，调用 `update_player_memory`。
    *   档案应采用 Markdown 格式，记录玩家的**兴趣偏好**、**性格标签**、**等级勋章**以及**历史贡献概括**。

## 2. 档案结构建议 (Markdown Format)

```markdown
# 玩家档案: [玩家名]
- **当前等级**: [C/B/A/S]
- **交互标签**: [正面/负面/中立]
- **关键细节**: [具体喜好、需求、坐标等]
- **里程碑**:
  - [日期]: [具体事件摘要]
```

## 3. 个性化应用

*   **人设渲染**：决策依据来自档案，但对话风格必须完全遵循 `SOUL.md` 的定义。
*   **连续性交互**：确保不同时段的回复能体现出你对该玩家过去经历的知识积累。
